## Audio: Xeno-canto (San Diego Subset)

| Property | Value |
|----------|-------|
| Source | https://xeno-canto.org |
| API | v3 (REST/JSON, API key required) |
| Format | MP3 (downloaded as-is; converted to WAV at preprocess time) |
| License | Creative Commons (varies per recording — CC BY-SA or CC BY-NC-SA) |
| Output path | `data/raw/xeno_canto/<SPECIES_CODE>/` |

### What we download

One directory per species (4-letter AOU code), containing:
- Up to 100 MP3 recordings per species filtered to quality grade A or B
- A `<CODE>_metadata.json` sidecar with full API response for every file

Quality grades (A = best):
- **A** — Full song, no or minimal background noise
- **B** — Good recording, some background noise acceptable
- C, D, E — Excluded from training

Geographic filter: United States, county-level filtering handled by species
range — Xeno-canto does not support county filtering via API.

### Setup

1. Register at https://xeno-canto.org (free, email verification required)
2. Navigate to your profile → API → Generate key
3. Add to your `.env` file:
   ```
   XENO_CANTO_API_KEY=your_key_here
   ```
4. Run the download script:
   ```bash
   python scripts/download_datasets.py
   ```

### Rate limiting

The script enforces:
- 1.0 second delay between API (metadata) requests
- 0.5 second delay between audio file downloads

Xeno-canto explicitly warns about IP blocks for aggressive scrapers.
Do not remove or reduce these delays.

## Visual: NABirds

| Property | Value |
|----------|-------|
| Source | https://dl.allaboutbirds.org/nabirds |
| Format | JPEG images, structured label files |
| License | Research use only — see NABirds Terms of Use |
| Output path | `data/raw/nabirds/` |

### Manual download required

NABirds cannot be downloaded programmatically — it requires agreeing to
Cornell Lab of Ornithology's terms of use via a web form.

**Steps:**
1. Go to https://dl.allaboutbirds.org/nabirds
2. Fill out the request form (name, institution, intended use)
3. Download the tarball (approx. 9GB)
4. Extract to `data/raw/nabirds/`:
   ```bash
   tar -xzf nabirds.tar.gz -C data/raw/
   ```

Expected directory structure after extraction:
```
data/raw/nabirds/
├── images/                     # ~48,000 images across 555 species
├── classes.txt                 # class_id → species name
├── hierarchy.txt               # taxonomic hierarchy
├── image_class_labels.txt      # image_id → class_id
├── images.txt                  # image_id → relative file path
├── train_test_split.txt        # image_id → 0 (test) or 1 (train)
├── bounding_boxes.txt          # image_id → x, y, w, h
├── sizes.txt                   # image_id → width, height
└── nabirds.py                  # helper script from Cornell
```

### Verifying the download

```bash
python scripts/download_datasets.py --audio-only  # skip audio, just verify NABirds
```

Or verify directly:
```python
from src.data.downloader import verify_nabirds
from pathlib import Path
print(verify_nabirds(Path("data/raw/nabirds")))  # True = good
```

### SD species subset

NABirds covers 555 North American species. For training, we filter to the
20 species in `configs/species.yaml`. This filtering happens at train time
in Phase 3 (`notebooks/visual_baseline.ipynb`) — the full NABirds dataset
is kept on disk so no re-download is needed if the species list expands.

## Train / Val / Test Splits

Splits are generated by `src/data/splitter.py` (Phase 2 data pipeline) and
written to `data/splits/` as CSV manifests:

| File | Contents |
|------|----------|
| `data/splits/audio_train.csv` | Paths + labels for audio training set |
| `data/splits/audio_val.csv` | Paths + labels for audio validation set |
| `data/splits/audio_test.csv` | Paths + labels for audio test set (held out) |
| `data/splits/visual_train.csv` | Paths + labels for visual training set |
| `data/splits/visual_val.csv` | Paths + labels for visual validation set |
| `data/splits/visual_test.csv` | Paths + labels for visual test set (held out) |

Split ratio: **60 / 20 / 20** (train / val / test), stratified by species.

CSV schema:
```
file_path,species_code,split
data/raw/xeno_canto/ANHU/ANHU_123456.mp3,ANHU,train
```

Splits are deterministic (fixed `random_seed` in `configs/thresholds.yaml`).

## Adding a New Species

1. Add the species entry to `configs/species.yaml` (code, common_name, scientific_name)
2. Re-run `python scripts/download_datasets.py --species <NEW_CODE>`
3. Re-generate splits: `python scripts/generate_splits.py` (Phase 2)
4. Update `docs/DATASETS.md` if the source or coverage changes


## Model Artifacts

Model weights are excluded from version control (`models/**/*.pt`, `models/**/*.pkl`).
After cloning, regenerate them by running the training notebooks in order:

| Artifact | Generated by | Notes |
|----------|-------------|-------|
| `models/audio/finetuned_sdbirds.pt` | `notebooks/audio_birdnet.ipynb` | Phase 4 CNN — not used for inference |
| `models/visual/finetuned_sdbirds.pt` | `notebooks/visual_efficientnet.ipynb` | Phase 4 fine-tuned — not used for inference |
| `models/visual/frozen_extractor.pt` | `notebooks/visual_efficientnet.ipynb` cell 28 | Phase 5 — required for VisualClassifier |
| `models/visual/sklearn_pipeline.pkl` | `notebooks/visual_efficientnet.ipynb` cell 28 | Phase 5 — required for VisualClassifier |

BirdNET weights are managed by the `birdnetlib` package and require no separate download.
